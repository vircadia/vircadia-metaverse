Project Apollo MetaverseAPI Implementation State

This lists each of the MetaverseAPI requests, a short description, and their implmentation and testing state.

Eventually there will be an OpenAPI (Swagger) definition for each request along with a detailed description of its operation. Pointers to these will eventually appear in the "Reference" column.

Domain Registration and Control

Description Implemented Tested Verified Reference
GET /api/v1/domains/{domainID}
Returns the ID, name, and ice-server address for a domain. Optionally the APIkey.
 ✔ ✔
PUT /api/v1/domains/{domainID}
Sets domain specific parameters. Considered the domain heartbeat and sent every 15 seconds.
 ✔ ✔
PUT /api/v1/domains/{domainID}/ice_server_address
Tells the metaverse-server which ICE server the domain is using.
 ✔ ✔
POST /api/v1/domains/temporary
Used by a new domain to get a temporary domain name and access APIkey
 ✔ ✔
GET /api/v1/domains/{domainID}/public_key
Fetch a domain's public key.
 ✔ ✔
PUT /api/v1/domains/{domainID}/public_key
Sets a domain's public key.
 ✔ ✔

Access Token Fetch and Update

Description Implemented Tested Verified Reference
POST /oauth/token
"Login" a user and return an access token. Only "password" grant type supported
 ✔ ✔
GET /user/tokens/new
Return a text HTML field showing a new token allocated to the logged in account. Only "for_domain_server" query accepted.
 ✔ ✔
GET /api/v1/token/new?scope=SCOPE
Return a new access token for the logged in account with the specified SCOPE. Scopes supported are "owner" and "domain".
 ✔ ✔
